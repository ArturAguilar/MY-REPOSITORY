ARIA (Accessible Rich Internet Applications)

ARIA (Accessible Rich Internet Applications) é um conjunto de atributos que podem ser adicionados aos elementos HTML para melhorar a acessibilidade de conteúdos interativos, especialmente aqueles criados com JavaScript, como menus, sliders, modais e outros componentes dinâmicos. A ARIA ajuda a fornecer informações extras aos leitores de tela e outros dispositivos assistivos, permitindo que os usuários com deficiência tenham uma experiência mais completa e acessível.

Por que usar ARIA?

O HTML padrão fornece algumas boas práticas de acessibilidade, mas com o uso de tecnologias mais complexas, como JavaScript, a estrutura padrão pode não ser suficiente para garantir que o conteúdo seja compreensível para todos. A ARIA ajuda a melhorar a interação com esses elementos dinâmicos, oferecendo informações adicionais sobre o comportamento e o estado desses elementos.

Exemplos de Atributos ARIA

aria-label

O atributo aria-label fornece uma descrição para um elemento quando o nome do próprio elemento não é suficiente ou não é claro. Ele é frequentemente usado quando há um botão ou ícone sem texto visível, como um ícone de "lupa" para pesquisa.

<button aria-label="Pesquisar">
<img src="search-icon.png" alt="">
</button>

Neste exemplo, o botão tem um ícone de "lupa", mas o aria-label garante que um leitor de tela entenda que ele é um botão de pesquisa.

aria-hidden

O aria-hidden="true" é usado para ocultar elementos da árvore de acessibilidade, ou seja, o elemento não será acessível por leitores de tela. Isso é útil quando você deseja que certos elementos visuais sejam ignorados, mas ainda assim visíveis para usuários não deficientes.

<div aria-hidden="true">
Este conteúdo será ignorado por leitores de tela.
</div>

aria-live

O aria-live indica que o conteúdo de uma região da página será atualizado dinamicamente e que o leitor de tela deve "escutar" essas mudanças. Isso é útil em situações de conteúdo dinâmico, como notificações ou mensagens de erro.

<div aria-live="polite">
A nova notificação foi recebida.
</div>

Neste caso, o conteúdo será anunciado assim que for alterado, mas de forma "polida", ou seja, não interrompendo o que o usuário está ouvindo no momento.

aria-role

O atributo aria-role define o papel de um elemento, como "botão", "caixa de diálogo" ou "navegação", para garantir que ele seja compreendido corretamente pelos leitores de tela.

<div role="button" aria-pressed="false">
Clique aqui
</div>

Aqui, o role="button" indica ao leitor de tela que esse div deve ser interpretado como um botão, mesmo que não seja um elemento button real.

aria-expanded

O aria-expanded é usado para indicar o estado de expansão ou colapso de um elemento interativo, como um menu ou uma árvore de navegação.

<button aria-expanded="false" aria-controls="menu">Menu</button>
<div id="menu" aria-hidden="true">Itens do menu</div>

No exemplo acima, o aria-expanded indica que o menu está fechado (false). Quando o usuário abrir o menu, o valor seria alterado para true, e o aria-hidden do menu mudaria para false, tornando-o acessível.

ARIA vs HTML Padrão

O HTML padrão já oferece várias maneiras de criar conteúdo acessível, por exemplo:

  • Usar a tag button para botões.
  • Usar nav para seções de navegação.
  • Usar img alt="Descrição da imagem" para imagens.

O ARIA, por sua vez, deve ser usado quando o HTML nativo não oferece as ferramentas necessárias para garantir a acessibilidade. A ARIA complementa o HTML, mas não substitui boas práticas de HTML. Ou seja, sempre que possível, você deve usar o HTML semântico padrão e adicionar ARIA apenas quando necessário, para garantir que o conteúdo dinâmico ou interativo seja acessível.

Exemplo de Integração entre ARIA e HTML

Aqui está um exemplo prático de como você pode combinar HTML semântico com ARIA para criar uma interface acessível.

<!DOCTYPE html>
<html lang="pt">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>Exemplo de Acessibilidade</title>
</head>
<body>

<nav>
<ul>
<li><a href="#home">Início</a></li>
<li><a href="#about">Sobre</a></li>
<li><a href="#contact">Contato</a></li>
</ul>
</nav>

<section id="home">
<h1>Bem-vindo ao nosso site!</h1>
<button aria-label="Fechar" aria-expanded="false" aria-controls="menu">
<img src="close-icon.png" alt="">
</button>
<div id="menu" aria-hidden="true">
<p>Menu com opções</p>
</div>
</section>

</body>
</html>

Considerações Importantes:

  • ARIA não deve ser usado como substituto para HTML semântico. Em vez de usar role="button" em um div, você deve usar o próprio elemento button.
  • Manter os valores atualizados: Quando você utiliza ARIA, deve garantir que os atributos estejam sempre atualizados com o estado atual do elemento. Por exemplo, aria-expanded deve ser alterado quando um menu é aberto ou fechado.
  • Testar a acessibilidade: Depois de adicionar ARIA, é importante testar o site com ferramentas de leitores de tela, como o NVDA, JAWS ou VoiceOver, para garantir que os elementos estejam funcionando corretamente.
