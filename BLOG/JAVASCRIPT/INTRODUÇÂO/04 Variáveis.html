Variáveis em JavaScript

Variáveis são contêineres que armazenam dados que podem ser alterados durante a execução de um programa. Elas permitem que os desenvolvedores armazenem, recuperem e manipulem valores de forma dinâmica.

Declaração de Variáveis:

Em JavaScript, as variáveis podem ser declaradas utilizando três palavras-chave principais: var, let e const.

var:

Introduzida nas versões iniciais do JavaScript, var tem escopo de função, o que significa que uma variável declarada com var é visível em toda a função onde foi definida.

Exemplo:

function exemploVar() {
var x = 10;
if (true) {
var x = 20; // mesma variável
console.log(x); // 20
}
console.log(x); // 20
}

Neste exemplo, a variável x dentro do bloco if é a mesma que fora dele, devido ao escopo de função do var.

let:

Introduzida no ECMAScript 6 (ES6), let tem escopo de bloco, ou seja, a variável é limitada ao bloco onde foi declarada.

Exemplo:

function exemploLet() { let x = 10; if (true) { let x = 20; // variável diferente console.log(x); // 20 } console.log(x); // 10 }

Aqui, a variável x dentro do bloco if é distinta da variável x fora do bloco, devido ao escopo de bloco do let.

const:

Também introduzida no ES6, const é usada para declarar constantes, ou seja, variáveis cujo valor não pode ser reatribuído após a inicialização.

Exemplo:

const PI = 3.14159;
PI = 3.14; // Erro: Assignment to constant variable.

No entanto, é importante notar que, embora o identificador não possa ser reatribuído, se a constante for um objeto ou array, suas propriedades ou elementos podem ser modificados.

Exemplo:

const pessoa = { nome: "João", idade: 30 };
pessoa.idade = 31; // Válido
pessoa = { nome: "Maria", idade: 25 }; // Erro: Assignment to constant variable.

Regras de Nomenclatura:

Ao nomear variáveis em JavaScript, siga estas regras:

  • Os nomes devem começar com uma letra, sublinhado (_) ou cifrão ($).
  • Os caracteres subsequentes podem ser letras, dígitos, sublinhados ou cifrões.
  • Nomes de variáveis são sensíveis a maiúsculas e minúsculas (idade e Idade são variáveis diferentes).
  • Evite usar palavras reservadas do JavaScript como nomes de variáveis (por exemplo, break, for, function, etc.).

Inicialização de Variáveis:

É uma boa prática inicializar variáveis no momento da declaração para evitar comportamentos inesperados.

Exemplo:

let nome = "João";
const idade = 30;

Hoisting:

Em JavaScript, declarações de variáveis e funções são "elevadas" ao topo de seu escopo durante a fase de compilação. No entanto, apenas a declaração é elevada; a inicialização permanece no local original.

Exemplo com var:

console.log(x); // undefined
var x = 5;

Aqui, a declaração var x; é içada para o topo, mas a inicialização x = 5; não é, resultando em undefined quando x é logado antes da inicialização.

Exemplo com let e const:

console.log(y); // ReferenceError: Cannot access 'y' before initialization
let y = 5;

Variáveis declaradas com let e const também são içadas, mas não podem ser acessadas antes de serem inicializadas, resultando em um erro de referência.

Conclusão:

Compreender as variáveis em JavaScript é fundamental para o desenvolvimento eficaz. A escolha entre var, let e const depende do escopo desejado e da mutabilidade da variável. Recomenda-se o uso de let para variáveis que podem mudar de valor e const para constantes ou referências que não devem ser reatribuídas. O uso de var é desencorajado em código moderno devido ao seu escopo de função e possíveis comportamentos inesperados.
