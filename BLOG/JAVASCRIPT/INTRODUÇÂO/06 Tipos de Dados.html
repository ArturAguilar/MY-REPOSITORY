Objetos em JavaScript

Em JavaScript, objetos são estruturas que armazenam pares chave-valor e permitem representar entidades do mundo real com propriedades e comportamentos.

Criando Objetos em JavaScript

Um objeto pode ser criado de várias formas:

Criando um Objeto Literal

A maneira mais simples de criar um objeto é usando chaves {}.

let pessoa = {
nome: "Artur",
idade: 18,
cidade: "São João de Meriti"
};

console.log(pessoa); // Exibe todo o objeto

✅ Vantagens das Template Strings: ✔️ Permitem quebras de linha sem \n.

✔️ Interpolação mais legível com \|.

console.log(pessoa.nome); // "Artur"

Cada par chave-valor dentro do objeto representa uma propriedade.


Criando um Objeto Usando new Object()

Outra forma de criar um objeto é utilizando o construtor Object().

let carro = new Object();
carro.marca = "Toyota";
carro.modelo = "Corolla";
carro.ano = 2024;

console.log(carro.marca); // "Toyota"

Mas a forma mais comum e recomendada é a criação via objeto literal {}.

Propriedades de um Objeto

As propriedades de um objeto são variáveis dentro do objeto, acessadas com ponto (.) ou colchetes ([]).

Acessando Propriedades

console.log(pessoa.nome); // "Artur"
console.log(pessoa["idade"]); // 18

Se a propriedade não existir, retorna undefined:

console.log(pessoa.altura); // Undefined


Adicionando e Modificando Propriedades

Podemos adicionar ou modificar propriedades de um objeto assim:

pessoa.profissao = "Desenvolvedor";
pessoa.idade = 19; // Modifica a idade

console.log(pessoa);


Removendo Propriedades

Usamos delete para remover propriedades.

delete pessoa.cidade;
console.log(pessoa);

Métodos de um Objeto

Um método é uma função dentro de um objeto.

let aluno = {
nome: "Lucas",
idade: 20,
saudacao: function() {
return `Olá, meu nome é ${this.nome}!`;
}
};

console.log(aluno.saudacao()); // "Olá, meu nome é Lucas!"

this refere-se ao próprio objeto.

Exibindo Objetos

Existem várias formas de exibir um objeto em JavaScript:

Exibindo no Console

console.log(pessoa);


Convertendo para String (JSON)

Se quisermos transformar o objeto em texto, usamos JSON.stringify():

let pessoaTexto = JSON.stringify(pessoa);
console.log(pessoaTexto); // {"nome":"Artur","idade":19,"profissao":"Desenvolvedor"}

Construtores de Objetos

Construtores são funções que criam múltiplos objetos com a mesma estrutura.

function Carro(marca, modelo, ano) {
this.marca = marca;
this.modelo = modelo;
this.ano = ano;
}

let carro1 = new Carro("Ford", "Fiesta", 2020);
let carro2 = new Carro("Honda", "Civic", 2023);

console.log(carro1.marca); // "Ford"
console.log(carro2.modelo); // "Civic"

Strings em JavaScript

Strings são sequências de caracteres usadas para representar texto em JavaScript. Elas podem ser escritas de três formas:

let texto1 = "Isso é uma string";
let texto2 = 'Isso também é uma string';
let texto3 = `Isso é uma string template`;

✅ Dica: Strings sempre devem estar entre **aspas simples ' ', aspas duplas " " ou crases `, dependendo do uso.

Métodos de Manipulação de Strings

JavaScript possui vários métodos úteis para trabalhar com strings.


length - Comprimento da String

A propriedade length retorna o número de caracteres de uma string.

let palavra = "JavaScript";
console.log(palavra.length); // 10


slice(start, end) - Extrai Parte da String

Retorna uma parte da string, começando em start e terminando antes de end.

let texto = "Hello, world!";
console.log(texto.slice(0, 5)); // "Hello"
console.log(texto.slice(-6)); // "world!" (começa de trás para frente)


substring(start, end) - Similar ao slice()

A diferença é que não aceita índices negativos.

console.log(texto.substring(7, 12)); // "world"


replace(old, new) - Substitui Texto

Substitui a primeira ocorrência de old por new.

let frase = "Eu amo JavaScript!";
console.log(frase.replace("JavaScript", "Python")); // "Eu amo Python!"

✅ Dica: Para substituir todas as ocorrências, use uma expressão regular com g:

let frase2 = "JavaScript é bom, JavaScript é incrível!";
console.log(frase2.replace(/JavaScript/g, "Python"));
// "Python é bom, Python é incrível!"


toUpperCase() e toLowerCase() - Maiúsculas e Minúsculas

let nome = "Artur";
console.log(nome.toUpperCase()); // "ARTUR"
console.log(nome.toLowerCase()); // "artur"


concat() - Junta Strings

Une duas ou mais strings.

let parte1 = "Olá, ";
let parte2 = "mundo!";
console.log(parte1.concat(parte2)); // "Olá, mundo!"

✅ Dica: O operador + também pode ser usado para juntar strings:

console.log(parte1 + parte2); // "Olá, mundo!"


trim() - Remove Espaços Extras

Remove espaços em branco no início e no final da string.

let entrada = " Texto com espaços ";
console.log(entrada.trim()); // "Texto com espaços"

Métodos de Busca em Strings

indexOf(substring) - Primeira Ocorrência

Retorna a posição da primeira ocorrência da substring ou -1 se não for encontrada.

let frase3 = "A vida é bela!";
console.log(frase3.indexOf("vida")); // 2
console.log(frase3.indexOf("JavaScript")); // -1 (não encontrado)


lastIndexOf(substring) - Última Ocorrência

let texto4 = "Eu amo JavaScript, porque JavaScript é incrível!";
console.log(texto4.lastIndexOf("JavaScript")); // 25

search(regex) - Busca com Expressões Regulares

console.log(texto4.search(/JavaScript/i)); // 7 (ignora maiúsculas e minúsculas)


match(regex) - Retorna Todas as Ocorrências

console.log(texto4.match(/JavaScript/g)); // ["JavaScript", "JavaScript"]


includes(substring) - Verifica se Contém um Texto

Retorna true se a substring estiver presente.

console.log(texto4.includes("amo")); // true

Templates Strings

Templates strings são strings dinâmicas usando crases `, permitindo a interpolação de variáveis.


Interpolação de Variáveis com ${}

let nome2 = "Artur";
let idade = 18;
let mensagem = `Meu nome é ${nome2} e eu tenho ${idade} anos.`;
console.log(mensagem);

✅ Vantagens das Template Strings: ✔️ Permitem quebras de linha sem \n.

✔️ Interpolação mais legível com ${}.

Números em JavaScript

Em JavaScript, os números são sempre do tipo "number" e podem ser representados de várias formas:

let inteiro = 42; // Número inteiro
let decimal = 3.14; // Número decimal (ponto flutuante)
let negativo = -10; // Número negativo
let cientifico = 1.5e6; // Notação científica (1.5 × 10⁶ = 1500000)

✅ Dica: Diferente de outras linguagens, JavaScript não diferencia inteiros de decimais.

Operações com Números

Você pode usar operadores matemáticos comuns:

let soma = 10 + 5; // 15
let subtracao = 10 - 5; // 5
let multiplicacao = 10 * 5; // 50
let divisao = 10 / 2; // 5
let resto = 10 % 3; // 1 (resto da divisão)
let potencia = 2 ** 3; // 8 (2³)

Métodos para Trabalhar com Números

JavaScript oferece vários métodos úteis para manipular números.

toString() - Converte Número em String

let num = 42;
console.log(num.toString()); // "42"
console.log((123).toString()); // "123"


toFixed(dígitos) - Formata Casas Decimais

Retorna um número com a quantidade de casas decimais especificada.

let pi = 3.1415926535;
console.log(pi.toFixed(2)); // "3.14"
console.log(pi.toFixed(4)); // "3.1416" (arredonda)

✅ Útil para exibir valores monetários.


toPrecision(dígitos) - Formata Precisão Total

Define o número total de dígitos a serem exibidos.

let num2 = 123.456;
console.log(num2.toPrecision(5)); // "123.46" (arredonda)
console.log(num2.toPrecision(3)); // "123"


valueOf() - Retorna o Valor Numérico Puro

let x = 100;
console.log(x.valueOf()); // 100


parseInt() - Converte String para Número Inteiro

console.log(parseInt("100")); // 100
console.log(parseInt("100.99")); // 100 (remove decimal)
console.log(parseInt("10 anos")); // 10 (pega só o número no começo)
console.log(parseInt("abc")); // NaN (não é número)


parseFloat() - Converte String para Número Decimal

console.log(parseFloat("100.99")); // 100.99
console.log(parseFloat("10.5 anos")); // 10.5
console.log(parseFloat("abc")); // NaN

Propriedades do Objeto Number

O objeto Number contém algumas constantes importantes:

Propriedade Valor
Number.MAX_VALUE Maior número possível (≈ 1.79E+308)
Number.MIN_VALUE Menor número positivo possível (≈ 5E-324)
Number.POSITIVE_INFINITY Infinito positivo
Number.NEGATIVE_INFINITY Infinito negativo
Number.NaN "Not a Number"

Exemplo:

console.log(Number.MAX_VALUE); // 1.7976931348623157e+308
console.log(Number.MIN_VALUE); // 5e-324
console.log(1 / 0); // Infinity
console.log(-1 / 0); // -Infinity
console.log(100 / "abc"); // NaN (Não é um número)

Diferença entre == e === com Números

✅ Em JavaScript, == não verifica o tipo, mas === verifica.

console.log(10 == "10"); // true (converte string para número)
console.log(10 === "10"); // false (tipos diferentes)

BigInt em JavaScript

O BigInt é um tipo de dado introduzido no ES11 (ECMAScript 2020) para representar números inteiros maiores do que Number.MAX_SAFE_INTEGER, que é 9.007.199.254.740.991 (aproximadamente 2^53 - 1).

Com BigInt, agora podemos trabalhar com números extremamente grandes sem perder precisão.

Criando um BigInt

Para criar um BigInt, basta adicionar um n ao final do número ou usar BigInt():

let numeroGigante = 9007199254740991n; // Adicionando 'n'
console.log(numeroGigante); // 9007199254740991n

let outroNumero = BigInt(9007199254740991); // Usando BigInt()
console.log(outroNumero); // 9007199254740991n

Diferença entre Number e BigInt

Os números normais (Number) só são seguros até 2^53 - 1, ou seja, até 9.007.199.254.740.991.
Acima disso, os cálculos podem perder precisão.

Exemplo de imprecisão com Number:

let numeroNormal = 9007199254740991; // Maior número seguro
console.log(numeroNormal + 1); // 9007199254740992
console.log(numeroNormal + 2); // 9007199254740992 (Erro! Deveria ser 9007199254740993)

✅ Com BigInt, o problema não ocorre!

let numeroSeguro = 9007199254740991n;
console.log(numeroSeguro + 2n); // 9007199254740993n

Operações com BigInt

O BigInt suporta as mesmas operações matemáticas de números normais:

let a = 12345678901234567890n;
let b = 98765432109876543210n;

console.log(a + b); // Soma
console.log(b - a); // Subtração
console.log(a * b); // Multiplicação
console.log(b / a); // Divisão (arredonda para baixo)
console.log(b % a); // Módulo (resto da divisão)

⚠️ Atenção: A divisão de BigInt sempre arredonda para baixo!

console.log(10n / 3n); // 3n (não é 3.33, pois BigInt não trabalha com decimais)

Misturando Number e BigInt

🚨 BigInt NÃO pode ser misturado diretamente com Number.

console.log(10n + 5); // ❌ ERRO! Não pode misturar tipos diferentes.

✅ Para operar entre Number e BigInt, converta um dos valores:

console.log(10n + BigInt(5)); // ✅ 15n
console.log(Number(10n) + 5); // ✅ 15

Comparação entre BigInt e Number

Você pode comparar BigInt e Number, pois a comparação é coerente:

console.log(10n > 5); // true
console.log(10n === 10); // false (tipos diferentes!)
console.log(10n == 10); // true (compara valores, ignora tipo)

⚠️ Dica:

  • === verifica tipo e valor, então 10n === 10 retorna false.
  • == converte os tipos automaticamente, então 10n == 10 retorna true.

Convertendo BigInt para String

Para exibir um BigInt como string, use .toString():

let big = 12345678901234567890n;
console.log(big.toString()); // "12345678901234567890"

Convertendo BigInt para Number

🚨 Cuidado ao converter BigInt para Number, pois pode perder precisão!

let big = 12345678901234567890n;
let num = Number(big);
console.log(num); // 1.2345678901234567e+19 (perda de precisão!)

Booleanos em JavaScript

O Boolean é um tipo de dado em JavaScript que pode ter apenas dois valores:

  • true (verdadeiro)
  • false (falso)

Ele é essencial para tomadas de decisão e para controlar o fluxo do código, como em estruturas condicionais e loops.

Criando Booleanos

Podemos definir variáveis booleanas diretamente com true ou false:

let isJavaScriptFun = true;
let isSkyGreen = false;

console.log(isJavaScriptFun); // true
console.log(isSkyGreen); // false

Ou usando comparações:

let resultado = 10 > 5; // Comparação retorna um booleano
console.log(resultado); // true

Valores que São Considerados "Falsos"

Em JavaScript, alguns valores são automaticamente convertidos para false quando avaliados em um contexto booleano. Estes valores são chamados de "Falsy Values":

  • false
  • 0
  • " " (string vazia)
  • null
  • undefined
  • NaN (Not a Number)

✅ Qualquer outro valor é considerado true (truthy)!

Exemplo de valores falsy:

console.log(Boolean(0)); // false
console.log(Boolean("")); // false
console.log(Boolean(null)); // false
console.log(Boolean(undefined));// false
console.log(Boolean(NaN)); // false

Exemplo de valores truthy:

console.log(Boolean(1)); // true
console.log(Boolean("Hello")); // true
console.log(Boolean([])); // true
console.log(Boolean({})); // true
console.log(Boolean(function(){})); // true

Booleanos em Estruturas Condicionais

Os booleanos são usados para tomar decisões no código, como em um if:

let idade = 18;

if (idade >= 18) {
console.log("Você é maior de idade.");
} else {
console.log("Você é menor de idade.");
}

Como idade >= 18 retorna true, a primeira mensagem será exibida.

Convertendo Valores para Booleanos

Podemos converter qualquer valor para um booleano usando a função Boolean():

console.log(Boolean(100)); // true
console.log(Boolean("Hello"));// true
console.log(Boolean(0)); // false
console.log(Boolean(null)); // false

Outra maneira comum de converter valores em booleanos é usando !! (dupla negação):

let nome = "Artur";
console.log(!!nome); // true (porque a string não está vazia)

Operadores Booleanos

Operador Lógico AND (&&)

Retorna true apenas se ambas as condições forem true:

console.log(true && true); // true
console.log(true && false); // false
console.log(5 > 3 && 10 > 5);// true (ambas são verdadeiras)


Operador Lógico OR (||)

Retorna true se pelo menos uma das condições for true:

console.log(true || false); // true
console.log(false || false); // false
console.log(5 > 10 || 10 > 5);// true (uma delas é verdadeira)


Operador de Negação (!)

Inverte o valor booleano:

console.log(!true); // false
console.log(!false); // true
console.log(!0); // true (0 é `falsy`, então `!0` é `true`)

Booleanos em Comparações

Os operadores de comparação sempre retornam um booleano:

console.log(10 > 5); // true
console.log(10 < 5); // false
console.log(10 === 10); // true
console.log(10 !== 10); // false
console.log(10 >= 10); // true
console.log(10 <= 9); // false

Booleanos em Ternários (Operador ? :)

O operador ternário é uma forma curta de escrever um if:

let idade = 20;
let status = (idade >= 18) ? "Adulto" : "Menor de idade";
console.log(status); // "Adulto"

Se idade >= 18 for true, retorna "Adulto", senão retorna "Menor de idade".

Undefined em JavaScript

Em JavaScript, undefined é um valor especial que indica que uma variável foi declarada, mas ainda não tem um valor atribuído.

Quando uma Variável é undefined?

Variável sem Atribuição

Se você declara uma variável sem inicializá-la, seu valor será undefined automaticamente:

let nome;
console.log(nome); // undefined


Acessando Propriedades Inexistentes

Se tentarmos acessar uma propriedade que não existe em um objeto, o JavaScript retorna undefined:

let pessoa = { nome: "Artur" };
console.log(pessoa.idade); // undefined (a propriedade 'idade' não existe)


Funções sem return

Se uma função não retorna nada explicitamente, ela retorna undefined por padrão:

function digaOi() {
console.log("Oi!");
}

let resultado = digaOi(); // A função apenas imprime, mas não retorna nada
console.log(resultado); // undefined


Parâmetros Opcionais

Se uma função for chamada sem fornecer todos os argumentos esperados, os parâmetros ausentes serão undefined:

function saudacao(nome) {
console.log("Olá, " + nome);
}

saudacao(); // Olá, undefined


undefined em Arrays

Se tentarmos acessar um índice que não existe em um array, o JavaScript retorna undefined:

let numeros = [10, 20, 30];
console.log(numeros[5]); // undefined (pois o índice 5 não existe)

Diferença entre undefined e null

Muitas vezes, undefined e null podem parecer iguais, mas há uma diferença importante:

undefined null
O valor não foi definido O valor foi definido como nulo propositalmente
O JavaScript atribui automaticamente O programador define manualmente
Exemplo: let x; Exemplo: let x = null;

let a; // undefined (JavaScript define automaticamente)
let b = null; // null (valor intencionalmente vazio)

Evitando undefined

✅ Sempre inicialize suas variáveis

Em vez de deixar uma variável sem valor:

let idade; // ❌ Pode causar problemas

Prefira sempre inicializá-la:

let idade = 0; // ✅ Melhor prática


✅ Use valores padrão com || ou ??

Se uma variável pode ser undefined, podemos definir um valor padrão usando || (OU lógico) ou ?? (nullish coalescing):

let nome;
console.log(nome || "Desconhecido"); // "Desconhecido" (se for undefined, usa esse valor)

Ou usando ??, que não substitui false e 0 (ao contrário do ||):

let nome = undefined;
console.log(nome ?? "Usuário Anônimo"); // "Usuário Anônimo"

Como Testar se Algo é undefined?

Podemos verificar se um valor é undefined de várias formas:

✅ Usando Comparação Direta

let x;
if (x === undefined) {
console.log("x é undefined");
}


✅ Usando typeof

A função typeof retorna "undefined" quando o valor é undefined:

let y;
console.log(typeof y); // "undefined"

Null em JavaScript

Em JavaScript, null é um valor especial que significa "ausência intencional de valor". Diferente de undefined, que indica uma variável não inicializada, null deve ser atribuído manualmente.

Quando null é Usado?

Atribuição Manual

Podemos atribuir null a uma variável para indicar que ela está vazia propositalmente:

let nome = null;
console.log(nome); // null


Representando a Ausência de Objeto

Se uma variável deveria armazenar um objeto, mas no momento não tem nenhum valor válido, podemos usar null:

let usuario = null; // Nenhum usuário ativo no momento


null em Comparações

null é um valor falso (falsy), o que significa que ele é tratado como false em expressões condicionais:

if (!null) {
console.log("null é considerado falso!"); // Será impresso
}

Como Testar se um Valor é null?

✅ Usando Comparação Direta

A forma mais comum de verificar se um valor é null é compará-lo diretamente com ===:

let x = null;
if (x === null) {
console.log("x é null");
}

⚠ Cuidado! Não use ==, pois null == undefined retorna true (mas null === undefined retorna false).

console.log(null == undefined); // true (evite)
console.log(null === undefined); // false (recomendado)


✅ Usando typeof

typeof não é muito útil para null, pois retorna "object", o que pode ser confuso:

console.log(typeof null); // "object" (isso é um bug histórico do JavaScript)

Por isso, prefira === null para verificar.

Como Usar null de Forma Correta?

✅ Inicializar Variáveis que Serão Atribuídas Depois

Se uma variável ainda não tem um valor, mas será preenchida depois, pode ser uma boa prática iniciá-la com null:

let resultado = null; // Será preenchido depois


✅ Apagar Dados Sem Excluir a Variável

Se quisermos "limpar" uma variável sem removê-la completamente:

let usuario = { nome: "Artur" };
usuario = null; // O objeto foi apagado, mas a variável ainda existe


✅ Evitar null Quando Possível

Em muitos casos, é melhor usar valores padrão em vez de null. Podemos usar || (OU lógico) ou ?? (nullish coalescing):

let nome = null;
console.log(nome || "Desconhecido"); // "Desconhecido" (se for null, usa esse valor)

Ou com ??, que não substitui false e 0 (ao contrário do ||):

let idade = null;
console.log(idade ?? "Idade não informada"); // "Idade não informada"

Symbol em JavaScript

O Symbol é um tipo de dado primitivo introduzido no ES6 (ECMAScript 2015). Ele é usado para criar valores únicos que podem ser utilizados como identificadores de propriedades de objetos, evitando conflitos entre nomes de propriedades.

Criando um Symbol

A forma mais básica de criar um Symbol é chamando a função Symbol():

let id = Symbol();
console.log(id); // Symbol()

Cada vez que Symbol() é chamado, um novo símbolo único é criado:

let id1 = Symbol();
let id2 = Symbol();

console.log(id1 === id2); // false (mesmo que pareçam iguais, são únicos)

Diferente de strings, dois símbolos nunca são iguais, mesmo que tenham a mesma descrição.

Symbol com Descrição

Podemos fornecer uma descrição opcional para um Symbol, apenas para depuração (o valor ainda é único):

let id = Symbol("id do usuário");
console.log(id); // Symbol(id do usuário)

Essa descrição não influencia a unicidade do Symbol, é apenas para referência.

Usando Symbol como Chave de Objeto

Podemos usar Symbol como chave de propriedade em objetos. Isso é útil para criar propriedades privadas, já que símbolos não aparecem em loops normais.

let id = Symbol("id");

let usuario = {
nome: "Artur",
[id]: 12345, // Symbol como chave
};

console.log(usuario);
// { nome: 'Artur', [Symbol(id)]: 12345 }

console.log(usuario[id]); // 12345

Symbol Não Aparece em for...in e Object.keys

Se tentarmos percorrer um objeto com um for...in ou Object.keys(), a propriedade Symbol não aparecerá:

for (let chave in usuario) {
console.log(chave); // Apenas "nome", o Symbol é ignorado
}

console.log(Object.keys(usuario)); // ["nome"]

Mas podemos acessar os Symbols de um objeto usando Object.getOwnPropertySymbols():

console.log(Object.getOwnPropertySymbols(usuario)); // [ Symbol(id) ]

Ou pegar todas as propriedades, incluindo as de Symbol, com Reflect.ownKeys():

console.log(Reflect.ownKeys(usuario)); // [ 'nome', Symbol(id) ]

Symbol.for() e Symbol.keyFor()

Normalmente, cada Symbol() criado é único, mas podemos criar Symbols globais compartilhados usando Symbol.for():

let id1 = Symbol.for("idGlobal");
let id2 = Symbol.for("idGlobal");

console.log(id1 === id2); // true (pois foram registrados no mesmo registro global)

Podemos recuperar a chave de um Symbol global com Symbol.keyFor():

console.log(Symbol.keyFor(id1)); // "idGlobal"

Isso só funciona para Symbol.for(), não para Symbol() normais.

Símbolos Bem Conhecidos (Well-Known Symbols)

O JavaScript tem alguns Symbols internos usados para modificar o comportamento de objetos e classes:

Símbolo Uso
Symbol.iterator Define como um objeto pode ser iterado com for...of
Symbol.toPrimitive Define como um objeto é convertido para primitivos (string, number, etc.)
Symbol.toStringTag Define a descrição retornada por Object.prototype.toString()
Symbol.hasInstance Controla o comportamento do operador instanceof

Quando Usar Symbol?

✅ Casos de Uso Ideais

✔ Criar propriedades privadas em objetos.
✔ Garantir que chaves de objetos sejam únicas.
✔ Evitar conflitos entre propriedades de bibliotecas diferentes.
✔ Usar Symbols globais para armazenar identificadores reutilizáveis.

Evitar Symbol Quando...

❌ Você precisa converter para string com frequência (pois Symbol não é convertível diretamente).
❌ Você quer que a propriedade apareça em JSON.stringify(), pois Symbol é ignorado.
